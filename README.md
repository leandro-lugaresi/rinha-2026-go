# Rinha de Backend 2026 — Fraud Detection with Vector Search

Entry for [Rinha de Backend 2026](https://github.com/zanfranceschi/rinha-de-backend-2026).  
A Go-based fraud detection API using vector search over a labeled reference dataset.

and implementation session is available at

## Endpoints

- `GET /ready` — returns `2xx` when the API is ready.
- `POST /fraud-score` — evaluates a transaction payload and returns the decision.

```json

{
  "id": "tx-123",
  "transaction": {

    "installments": 3,

  },
  "customer": {
    "avg_amount": 769.76,
    "tx_count_24h": 3,
    "known_merchants": ["MERC-001"]
  },
  "merchant": {
    "id": "MERC-001",
    "mcc": "5912",
    "avg_amount": 298.95
  },
  "terminal": {
    "is_online": false,
    "card_present": true,
    "km_from_home": 13.7
  },
  "last_transaction": {
    "timestamp": "2026-03-11T14:58:35Z",
    "km_from_current": 18.8
  }

```

```json
// Response
{
  "approved": false,
  "fraud_score": 0.8
}
```

## Architecture

- **Load Balancer:** HAProxy 2.9 (round-robin, `/ready` health checks)
- **API:** 2 instances of a Go service (port 9999)
- **Search:** IVF (Inverted File Index) with pre-quantized uint8 vectors for approximate nearest-neighbor search
- **Reference data:** 3M vectors pre-processed at build time into a compact binary IVF index
- **Resource limits:** 1.0 CPU total, 340 MB memory total (HAProxy 0.10 CPU / 20 MB; API 0.45 CPU / 160 MB each)

## Tech Stack

- Go 1.26
- HAProxy 2.9
- Docker + Docker Compose
- IVF index with 8192 partitions and uint8 quantization

## How to run locally

```bash
# Build and start the stack
docker compose up -d

# Wait for /ready
curl -f http://localhost:9999/ready

# Send a test transaction
curl -sf -X POST http://localhost:9999/fraud-score \
  -H 'Content-Type: application/json' \
  -d '{
    "id": "smoke-001",
    "transaction": { "amount": 100, "installments": 3, "requested_at": "2026-03-11T20:23:35Z" },
    "customer": { "avg_amount": 200, "tx_count_24h": 3, "known_merchants": ["MERC-001"] },
    "merchant": { "id": "MERC-001", "mcc": "5912", "avg_amount": 150 },

    "last_transaction": { "timestamp": "2026-03-11T14:58:35Z", "km_from_current": 18.8 }
  }'
```

## Submission

The `submission` branch is automatically generated by a GitHub Actions workflow:
- Builds a `linux/amd64` image
- Publishes to `ghcr.io/leandro-lugaresi/rinha-2026-go`
- Generates a clean `docker-compose.yml` pointing to the published image
- Force-pushes the `submission` branch
